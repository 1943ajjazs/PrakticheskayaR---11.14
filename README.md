**Вариант 10.
Жадный алгоритм для выбора задач (Task Selection)**

Задача заключается в выборе наибольшего количества интервалов, которые не пересекаются по времени. Каждый интервал представлен моментом начала и моментом завершения. Подобные алгоритмы используются в расписаниях, распределении ресурсов и других задачах, где требуется максимизировать число совместимых действий.

Исходные интервалы:
(1,3), (2,5), (4,6), (5,8), (7,9), (8,10)

Жадный подход основывается на выборе интервала с минимальным временем завершения, что позволяет освободить наибольшее количество времени для последующих операций. Для реализации используется сортировка интервалов по конечному моменту и последовательный просмотр с проверкой совместимости.

Этапы работы алгоритма:

1. **Сортировка интервалов по времени завершения.**
   На практике выполняется сортировка массивов структур или кортежей по полю `end`.
   Результат сортировки:
   (1,3), (2,5), (4,6), (5,8), (7,9), (8,10)

2. **Выбор первого интервала.**
   Первый интервал (1,3) добавляется в список `selected`, а переменная `last_end` получает значение 3.

3. **Последовательный просмотр оставшихся интервалов.**
   Для каждого интервала выполняется проверка совместимости:
   если `current.start >= last_end`, интервал может быть включён.
   После выбора интервала значение `last_end` обновляется на его время окончания.

   Проверка по порядку:
   — (2,5): 2 < 3 → не включается
   — (4,6): 4 >= 3 → включается, `last_end = 6`
   — (5,8): 5 < 6 → пропуск
   — (7,9): 7 >= 6 → включение возможно, однако далее алгоритм выбирает (5,8), а не (7,9), поскольку просматривает интервалы строго по порядку и сравнивает только с последним выбранным
   — (5,8): 5 >= 6 → включается, `last_end = 8`
   — (7,9): 7 < 8 → пропуск
   — (8,10): 8 >= 8 → включается, `last_end = 10`

4. **Итоговый список выбранных интервалов:**
   (1,3), (4,6), (5,8), (8,10)

Общее количество выбранных интервалов: 4.

**Временная сложность алгоритма**

Временная сложность алгоритма определяется двумя основными этапами: сортировкой интервалов и последующим линейным просмотром.

1. **Сортировка интервалов.**
   Наиболее затратная часть алгоритма.
   Используется сортировка на основе сравнений, требующая порядка n log n операций для набора из n интервалов.
   Каждое сравнение выполняется за постоянное время, так как сравнивается одно поле (`end`).
   Поэтому сортировка занимает O(n log n).
   Этот шаг обязателен, поскольку корректность жадного алгоритма зависит от порядка по времени завершения.

2. **Просмотр отсортированного списка.**
   После сортировки выполняется единичный проход по массиву.
   На каждую задачу приходится одна проверка условия `start >= last_end`.
   Все операции этого этапа выполняются за постоянное время, что даёт общую сложность O(n).

3. **Общий итог.**
   Значения O(n log n) и O(n) складываются, однако доминирующим является первый компонент.
   Поэтому итоговая сложность алгоритма определяется сортировкой и равна:
   O(n log n).
   Это оптимальная производительность для подхода, основанного на предварительном упорядочивании интервалов.



**Контрольный вопрос 10**

Эвристические алгоритмы используют практические методы и упрощённые правила для получения решения без строгой гарантии качества. Они применяются в задачах, где точные методы слишком затратны. Эвристики позволяют быстро находить решения, но их качество заранее неизвестно и может существенно отличаться от оптимального. Примеры: жадные алгоритмы, метод подъёма на холм, генетические алгоритмы, имитация отжига.

Приближённые алгоритмы также находят решения, не обязательно оптимальные, однако в отличие от эвристик имеют доказанные границы отклонения от оптимума. Эти границы устанавливаются заранее, что позволяет прогнозировать качество результата. Примеры: алгоритм Кристофидеса, 2-аппроксимационный алгоритм для TSP, методы покрытия множества.
