# PrakticheskayaR---11.14

**Вариант 10.
Жадный алгоритм для выбора задач (Task Selection)**

Задача состоит в выборе максимального количества интервалов, которые не пересекаются по времени. Каждый интервал задаётся моментом начала и завершения. Такой тип задачи часто встречается в планировании расписаний, распределении ресурсов и в системах, где требуется максимизировать число независимых действий в фиксированном промежутке времени.

Исходные интервалы:
(1,3), (2,5), (4,6), (5,8), (7,9), (8,10)

Жадный подход в данном случае основан на следующем принципе:
выбор интервала, который завершается раньше остальных, позволяет сохранить максимальное количество свободного времени для последующих интервалов. Поэтому алгоритм сначала упорядочивает все задачи по времени завершения, а затем последовательно выбирает те из них, которые не пересекаются с уже выбранными.

Этапы работы алгоритма:

1. Сортировка интервалов по времени окончания.
   После сортировки порядок остаётся таким же, поскольку исходный набор уже расположен в нужной последовательности:
   (1,3), (2,5), (4,6), (5,8), (7,9), (8,10)

2. Выбор первого интервала.
   Интервал (1,3) заканчивается раньше всех, поэтому он включается в решение.

3. Последующий просмотр интервалов.
   На этом шаге каждый интервал сравнивается с последним выбранным. Если начало текущего интервала не раньше момента окончания предыдущего выбранного интервала, то он добавляется в итоговую выборку.
   Просмотр выполняется в таком порядке:

   * (2,5) пересекается с (1,3), поэтому не включается;
   * (4,6) не пересекается, так как начинается после 3, добавляется;
   * (5,8) начинается после окончания (4,6), также добавляется;
   * (7,9) пересекается с интервалом (5,8), поэтому пропускается;
   * (8,10) начинается в момент, совпадающий с окончанием предыдущего выбранного интервала (5,8), что допускается, поэтому включается.

4. Итоговый набор интервалов:
   (1,3), (4,6), (5,8), (8,10)

Таким образом, максимальное количество непересекающихся интервалов составляет 4.
Жадный алгоритм обеспечивает корректный результат для данного типа задачи благодаря тому, что выбор на каждом шаге локально оптимален и не нарушает глобальную цель — максимизацию числа интервалов.

Временная сложность алгоритма определяется сортировкой интервалов и составляет O(n log n). Последующий однократный проход по списку имеет линейную сложность, поэтому дополнительного влияния не оказывает.

**Контрольный вопрос 10.
Эвристические алгоритмы. Отличия эвристических алгоритмов от приближённых. Примеры.**

Эвристические алгоритмы представляют собой методы, которые используют практические наблюдения, упрощённые правила и различные интуитивные стратегии для получения решения. Эвристики обычно применяются в ситуациях, когда точные алгоритмы слишком медленны или недоступны. Такие методы направлены на получение приемлемых решений за ограниченное время, однако они не дают формальных гарантий качества результата. Эвристики особенно распространены в задачах комбинаторной оптимизации, где пространство всех возможных решений очень велико.

Примеры эвристических методов:
— жадные алгоритмы;
— алгоритм ближайшего соседа в задаче коммивояжёра;
— метод подъёма на холм (Hill Climbing);
— генетические алгоритмы;
— имитация отжига.

Приближённые алгоритмы, в отличие от эвристических, имеют формальные гарантии качества решения. Для таких алгоритмов заранее известна величина, на которую найденное решение может отличаться от оптимального. Приближённые методы разрабатываются таким образом, чтобы обеспечить определённый уровень точности при приемлемом времени работы. Эти алгоритмы особенно важны для NP-трудных задач, где поиск точного оптимума требует экспоненциального времени.

Примеры приближённых алгоритмов:
— алгоритм Кристофидеса для метрической задачи коммивояжёра;
— 2-аппроксимационный алгоритм для той же задачи;
— алгоритмы для задачи покрытия множества;
— различные методы для задач составления расписаний.

Основное отличие между классами методов заключается в наличии или отсутствии гарантии на качество решения. Эвристические алгоритмы могут давать как хорошие, так и неудовлетворительные результаты и не обеспечивают строгих границ погрешности. Приближённые алгоритмы всегда сопровождаются математическими оценками, которые ограничивают отклонение от оптимального решения.
